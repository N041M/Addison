Brand & UI guidelines · v1 · July 2026

Addison

A local-first agent that anyone can trust. The interface reads like calm correspondence: paper surfaces, a fern accent, serif conversation, and one honest rule — blocky things are live annotations; rounded things are yours to own or act on.

Principles

Asks first. Consent is one calm question with two answers — never a modal, never jargon.

Shows its work. Every step is listed in plain words while it happens.

Everything undoes. Undo is always one visible click away.

One surface, two depths. Simple and Developer differ in what's shown, never in safety.

Logo — the service bell

Ring, and Addison comes — it acts when asked, never uninvited. Three shapes (knob, dome, base), fern on tint or cream on pine, monochrome only, never with effects. Lockup: bell + "Addison" in Source Serif 4 (500). See the logo sheet for variants and small sizes.

Color

Warm paper neutrals and one fern-green accent. Green is reserved for Addison's voice, primary actions, and live state — never decoration. Deliberately distant from vendor branding: no cream/terracotta, no purple, no steel blue.

Dark mode inverts value, not hue: same fern family, lightened for contrast; primary buttons flip to dark text (#14201A) on light fern. Body text is AA at 15px in both modes.

Typography

Source Serif 4 — the conversation

Messages, greetings, section titles. 17px/1.7 for message text; 500 weight for display. The serif is what makes threads read as correspondence, not chat.

Public Sans — the interface

Everything interactive: buttons, labels, sidebar, settings. 15px body, 13px controls, 10.5px small-caps section labels (letter-spacing .09–.11em, uppercase, faint).

IBM Plex Mono — the machine

Numbers and system facts only: token counts, latency, model tags, Developer diagnostics. Never for prose.

Shape language

The blocky/rounded mix is systematic, not decorative:

Blocky = live annotation

Square edges, 2px left rules, small-caps labels. Used for: "Addison's work" step lists, active sidebar item, section labels. The rule spans only its list — it ends where the list ends.

Rounded = ownable / actionable

Radius 10px cards, 999px pills. Used for: widgets, consent cards, composer, buttons, setup banner (12px). If the user can click it, run it, or keep it — it's rounded.

Radius scale: 6 small buttons · 8 inputs/rows · 10 cards · 12 banners · 999 pills. Shadows near-zero: 0 1px 4px rgba(34,38,31,.05); the pine banner alone gets a soft lift.

Layout & components

Three columns: sidebar 216px (chats, Settings, profile) · conversation ≤580px, independently scrolling · widget rail 270px, independently scrolling, hideable («/» toggle), overflow behind a stacked-edge tray tab.

Messages: no bubbles. Small-caps sender label (YOU faint / ADDISON fern) over serif text. Hover reveals "Rewind to here".

Consent card: fern-tint, rounded, one question ("Allow Addison to save…?"), one plain consequence line ("One new file. You can undo it afterwards."), pill Allow + text Not now. Rendered in the rail near the step it blocks; inline in-thread when the rail is hidden.

Widgets: user-pinned support scripts (routines, token meter, connections, anything Addison codes on request). Card anatomy: small-caps title, mono value, one optional action pill.

Setup banner: pine #1E2B25, serif headline, numbered steps, cream button. The only high-contrast block in the app — shown only in intro mode (no key, no local model) and never again after.

Voice: plain sentences, no jargon, no exclamation marks, always name the consequence and the way back ("You can undo this afterwards"). Errors are one sentence plus a next step, never a code.

Dark mode: add the dark palette under a .dark class (Tailwind darkMode:"class"), toggled from Settings → Appearance and persisted like addison.defaultRole in localStorage.

Fonts under the strict CSP: don't hotlink Google Fonts — bundle the three families as woff2 in shell/src/assets/fonts/ with @font-face in styles.css (all are OFL-licensed). Keep the system stack as fallback.

2 · Component moves

App.tsx — replace header+drawer shell with the 3-column grid: new Sidebar.tsx (conversation list needs a conversations table in the SQLite schema — currently single-thread), main pane router (chat | settings), WidgetRail.tsx.

ChatThread.tsx — keep the correspondence rows; swap message font to serif 17/1.7; composer becomes the rounded card (textarea + model pill left, Send right).

ActivityPanel.tsx → "Addison's work" block in the rail: same tool.activityUpdate feed, rendered as the ruled dot list (filled = done, outlined = in progress); "Save these steps as a routine" keeps calling proposeRoutine.

PermissionCard.tsx — restyle to the fern-tint rounded card; render in the rail when open, inline in-thread when the rail is hidden. Same respondToPermission wiring.

SettingsDrawer.tsx → SettingsPage.tsx: same sections (key, where-Addison-thinks, LocalModelSetup, RoutineLibrary, Profile) laid out as the 2×2 card grid, routed in-window — delete the scrim/translate drawer logic.

New: WidgetRail.tsx — pinned widget list + tray overflow + rail collapse (persist both in localStorage). Widgets are declarative JSON (id, title, valueSource, action?) so the core can propose new ones the same way it proposes routines: gated, undoable, saved only on confirm. Token/latency widgets read from a new stats.get IPC method; connections widget reuses role/status data the shell already has.

Setup banner — render in-thread when no role is configured (roles.every(r => !r.configured)); "Start setup" hands off to the existing Setup Assistant conversation; never shown again once a key or local model exists.

3 · Order of work

Tokens + fonts (pure config, everything reskins at once) → settings page (moves the drawer, no new data) → 3-column shell + sidebar (needs conversations schema) → rail with Addison's-work + consent (rewires existing feeds) → widgets + stats IPC → dark mode → first-run banner. Each step ships usable on its own; safety invariants (gate, undo, key isolation) are untouched throughout.
